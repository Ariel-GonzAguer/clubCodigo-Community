*3* Introducción a Code Review/Revisión de código :15 minutos.
- 1. Introducción: 
-> ¿Qué es una code review?
    --> Es un proceso donde los desarrolladores revisan el código escrito por otros, por lo general antes de integrarlo al proyecto.
-> ¿Por qué es importante?
    --> Mejora la calidad del código y previene errores.
    --> Promueve el aprendizaje compartido y asegura estándares del equipo.
    --> Reduce el tiempo de depuración a futuro.
- 2. Buenas prácticas clave:
   --> Enfocarse en un objetivo a la vez. Por ejemplo: ¿El código es funcional? ¿Es legible? ¿Cumple con los estándares?
   --> Ser constructivo. Usar un tono positivo y explicando las sugerencias. En lugar de "Esto está terrible", escribe "¿Qué opina si usamos [x método] para mejorar el rendimiento?".
   --> Verifica lo esencial. 
      ----> Funcionalidad: ¿El código hace lo que se espera?
      ----> Legibilidad: ¿Es fácil de entender? ¿Se usan nombres descriptivos?
      ----> Pruebas: ¿Incluye pruebas suficientes?
- 3. Demostración y práctica:
   --> Revisión del código en imágenes.
